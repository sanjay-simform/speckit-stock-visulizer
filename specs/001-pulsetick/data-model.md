# Data Model: PulseTick

**Feature**: PulseTick | **Date**: 2026-05-07 | **Related**: [plan.md](./plan.md)

## Core Entities

### Tracker

Represents a single asset being tracked in realtime. Each tracker maintains independent simulation state.

**Fields**:

- `id: string` ‚Äî UUID, unique identifier for this tracker instance
- `symbol: string` ‚Äî User-provided symbol name (TATA, BTC, INFY, etc.), max 10 characters
- `threshold: number` ‚Äî Volatility magnitude 0-100; controls price change percentage per tick
- `currentValue: number` ‚Äî Latest simulated price; always ‚â• 1
- `paused: boolean` ‚Äî Whether simulation is paused for this tracker; default false
- `history: number[]` ‚Äî Circular buffer of price history; max 60 points
- `createdAt: number` ‚Äî Timestamp when tracker was created (Date.now())

**Invariants**:

- `symbol` must be unique per server instance (no duplicates)
- `threshold` must be in range [0, 100]
- `currentValue` must be positive (minimum 1 to prevent negative prices)
- `history.length` must be ‚â§ 60 (circular buffer enforcement)
- `history[history.length - 1]` equals `currentValue` at all times

**Lifecycle**:

1. Created via `SimulationEngine.createTracker(symbol, threshold)`
2. Enters simulation loop (generates tick every second if not paused)
3. Can be paused/resumed without losing state
4. Can be removed (deleted from registry and stopped from simulation)

### Tick

Represents a single price update event. Ticks are generated by the simulation engine once per second per active tracker.

**Fields**:

- `id: string` ‚Äî UUID, unique identifier per tick (for deduplication)
- `symbol: string` ‚Äî Which tracker this tick belongs to (references Tracker.symbol)
- `value: number` ‚Äî New price value for this tick
- `delta: number` ‚Äî Change from previous value (value - previousValue)
- `timestamp: number` ‚Äî When this tick was generated (Date.now())

**Usage**:

- Sent from server to all connected clients via WebSocket broadcast
- Client uses tick to update UI (display new value, append to history, change colors)
- Delta used to determine color (green if positive, red if negative)

**Invariants**:

- `id` must be globally unique (UUID)
- `symbol` must match an active Tracker
- `value` must be positive
- `delta` may be negative or positive
- `timestamp` must be recent (within last few seconds)

### Point (Graph Point)

Derived from Tick; represents a single point on the graph visualization.

**Fields**:

- `x: number` ‚Äî Timestamp (used for X-axis, time)
- `y: number` ‚Äî Price value (used for Y-axis, price)
- `delta: number` ‚Äî Change from previous (used for color determination)

**Usage**:

- Array of Points rendered as line chart via Chart.js
- Frontend maintains rolling array of recent 60 Points per tracker
- Points automatically shifted out as new points arrive (circular buffer)

**Rendering Rules**:

- Green color if `delta >= 0`
- Red color if `delta < 0`
- Line remains continuous (Chart.js connects points)
- X-axis represents time progression (right edge = most recent)

```

  ‚îî‚îÄ‚îÄ Points derived from history + delta values
```

## State Transitions

### Tracker Lifecycle

```
CREATED
  ‚Üì (every 1 second)
ACTIVE ‚Üí generates Tick
  ‚Üì (user clicks pause)
PAUSED ‚Üí no Tick generation
  ‚Üì (user clicks resume)
ACTIVE
  ‚Üì (user removes)
REMOVED (deleted from simulation)
```

### Tick Generation

```
For each ACTIVE tracker per second:
  1. Calculate: changePercent = (random - 0.5) * 2 * (threshold / 100)
  2. Calculate: newValue = max(1, currentValue * (1 + changePercent))
  3. Calculate: delta = newValue - currentValue
  4. Update: tracker.currentValue = newValue
  5. Update: tracker.history.push(newValue); if length > 60, shift()
  6. Create: Tick { id, symbol, value, delta, timestamp }
  7. Broadcast: send Tick to all connected clients
```

## Validation Rules

**Tracker Creation**:

- Symbol must not be empty
- Symbol must not already exist (no duplicates)
- Threshold must be 0-100 (inclusive)

**Tracker Mutation**:

- Pause/resume on existing tracker ID
- Remove existing tracker ID
- Cannot change symbol or threshold after creation

**Tick Generation**:

- Only ACTIVE (non-paused) trackers generate ticks
- Tick value must be ‚â• 1 (enforced by max(1, ...))
- Delta must be accurate (current - previous)
- Timestamp must be current

## Memory Model

**Per Tracker**:

- `id`: 36 bytes (UUID string)
- `symbol`: ~10 bytes (max 10 chars)
- `threshold`: 8 bytes (number)
- `currentValue`: 8 bytes (number)
- `paused`: 1 byte (boolean)
- `history`: 60 \* 8 = 480 bytes (60 numbers)
- `createdAt`: 8 bytes (number)
- **Total: ~550 bytes per tracker**

**Scaling Example**:

- 100 trackers = 55 KB
- 1000 trackers = 550 KB
- Acceptable for MVP (<10MB heap usage)
